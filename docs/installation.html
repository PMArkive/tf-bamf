Installation

There are two ways to install BAMF:

  1. Use the installer - TBD
  2. Manual installation - See below

If you haven't downloaded BAMF yet, you can find the latest version on TF2Maps. For the full project and in-development versions, check GitHub.
Using either installation method, each module can be installed or left out individually. Note that some modules depend on each other. For information on what each individual module does, check the Contents & Features page.

VPK Module

The "VPK" module currently consists of two .vpk files and a loose folder. This is because the loose folder (Extra Materials) needs to be packed into the map, and CompilePal currently cannot read VPKs. This makes packing assets into VPKs a consistent method of assuring game files don't accidentally end up inside your map file.

  1. Installation:
    Paste the content of tf-bamf/VPKs/ into Team Fortress 2/tf/custom/.
    A correct path would look like Team Fortress 2/tf/custom/BAMF FGD Assets.vpk or Team Fortress 2/tf/custom/BAMF Extra Materials/materials/tools/toolshint.vmt.
    See the Contents & Features page for more information on what each sub-module contains.
  2. Usage:
    You can find the new materials in your material browser in Hammer. Materials in the Extra Materials sub-module will need to be packed into a map using CompilePal or shared with your collaborators.

FGD Module with Radshadowman and Propper support

Note that this module requires the BAMF FGD Assets sub-module from the VPK Module.

  1. Installation:
    Paste the content of the tf-bamf/FGD/ directory into Team Fortress 2/bin/.
    Do NOT paste it into Team Fortress 2/tf/bin/.
    Do NOT paste it into Team Fortress 2/bin/x64/.
    In Hammer, open Tools → Options → Game Configurations, remove all existing Game Configurations, then add tf-brokk.fgd as the only new one. You do not need a separate Propper FGD to be able to use Propper.
  2. Configuration: (Optional)
    Open Team Fortress 2/tf/bin/tf-bamf.fgd in the text editor of your choice and adjust the options at the very start of the document. You can enable or disable the lines in the entity window there, among other things.
  3. Usage:
    Once installed, the FGD will work out of the box. You will notice more and improved entity keyvalues and descriptions in Hammer. See the Textureshadows page on how to use RadShadowMan to automatically enable textureshadows on all static props in your map.

Prefab Module

  1. Installation:
    Drag the content of the tf-bamf/Prefabs/ directory into Team Fortress 2/bin/Prefabs/. If that directory does not exist, create it. Make sure your Prefab Directory is configured correctly in Hammer/Hammer++ under Tools → Options → Game Configurations → Prefab Directory.
  2. Usage:
    In Hammer, select the Entity Tool. Select the category of prefab you want to place in the Categories dropdown menu in the bottom right. Select the prefab you want to place in the Objects dropdown menu just below. Place the prefab by clicking into the 3D Viewport, just like you would when placing a point entity. Move and rotate as usual.
    All prefabs are grouped to make placement easier. Stay in Group Select mode to move the whole thing at once and press CTRL+B to align to the grid, if necessary. Most prefabs have a skip brush surrounding them for that purpose, which can be deleted once the prefab is placed correctly.

Game Mode Module

  1. Installation:
    Drag the content of the tf-bamf/Gamemodes directory anywhere you want.
  2. Usage:
    You can open the VMFs in Hammer just like any other VMF. Copy what you need and paste into your map. Some game modes have additional files that you need to place properly.

VMF Module

  1. Installation:
    Drag the content of the tf-bamf/VMFs/ directory anywhere you want.
  2. Usage:
    You can open the VMFs in Hammer just like any other VMF.

Compile Configuration Module - CompilePal

  1. Installation:
    Drag the content of tf-bamf/Compile Configs/CompilePal/ into the directory that contains your CompilePalX.exe.
  2. Usage:
    Compile your map using the BAMF TF2 Super Duper preset in CompilePal.

Tool - RadShadowMan

This requires the Compile Configuration Module for use with CompilePal or the FGD Module for use with Hammer++ or for you to create your own compile configuration running RadShadowMan.

  1. Installation:
    Drag Tools/Radshadowman/RadShadowMan.exe into Team Fortress 2/bin/.
  2. Usage:
    Compile your map using the RadShadowMan configuration in either Hammer++ or CompilePal.

Tool - Propper

This requires the Compile Configuration Module for use with CompilePal or the FGD Module for use with Hammer++ or for you to create your own compile configuration running Propper.

  1. Installation:
    Drag Tools/Propper/propper.exe into Team Fortress 2/bin/.
  2. Usage:
    Compile your map using the Propper configuration in either Hammer++ or CompilePal.
